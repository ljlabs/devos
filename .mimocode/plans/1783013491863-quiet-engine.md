# Plan: IDE File Browser Actions + Editor Path Display

## Goal
Add file/folder management (create, rename, delete, copy path) to the file browser via a context menu, and show the full selectable file path in the editor header.

---

## Feature 1: File Browser Context Menu

### 1A. Server — new file operations in `server_src/files.ts`

Add three exported functions after `writeFile()`:

- **`createEntry(workspaceRoot, relativePath, type: "file" | "directory")`**
  - Validate via `resolveWithinWorkspace`
  - For "file": `fs.writeFileSync(absPath, "", "utf-8")`
  - For "directory": `fs.mkdirSync(absPath, { recursive: true })`
  - Return `FileEntry`

- **`renameEntry(workspaceRoot, oldRelativePath, newName)`**
  - Validate both old and new paths via `resolveWithinWorkspace`
  - `fs.renameSync(oldAbs, newAbs)`
  - Return the new `FileEntry`

- **`deleteEntry(workspaceRoot, relativePath)`**
  - Validate via `resolveWithinWorkspace`
  - `fs.rmSync(absPath, { recursive: true, force: true })`

### 1B. Server — new API routes in `server_src/server.ts`

Import new functions, add routes after the existing `files/write` route:

| Route | Method | Body | Description |
|-------|--------|------|-------------|
| `POST /api/workspaces/:workspaceId/files/create` | POST | `{ path, type }` | Create file or directory |
| `POST /api/workspaces/:workspaceId/files/rename` | POST | `{ oldPath, newName }` | Rename file or directory |
| `DELETE /api/workspaces/:workspaceId/files/delete` | POST | `{ path }` | Delete file or directory |

All routes follow existing patterns: look up workspace, call `files.ts` function, return result.

### 1C. Frontend — context menu in `FileExplorer.tsx`

Add a state-driven context menu to `FileTreeItem`:

- **Trigger**: A small `⋯` (MoreVertical icon) button appears on the right side of each row on hover. Clicking it opens a floating dropdown menu positioned near the button.
- **Menu options**:
  - **New File** — opens inline rename-style input; on confirm, calls `POST /files/create` with `type: "file"`, refreshes parent directory
  - **New Folder** — same as New File but `type: "directory"`
  - **Rename** — inline input pre-filled with current name; on confirm, calls `POST /files/rename`
  - **Delete** — confirmation prompt then calls `DELETE /files/delete`
  - **Copy Path** — copies `entry.path` to clipboard via `navigator.clipboard.writeText()`
- **Click-outside** closes the menu
- The context menu is a new local component `FileContextMenu` within `FileExplorer.tsx`

New props added to `FileExplorerProps` and `FileTreeItemProps`:
- `onCreateEntry: (parentPath: string, name: string, type: "file" | "directory") => Promise<void>`
- `onRenameEntry: (oldPath: string, newName: string) => Promise<void>`
- `onDeleteEntry: (path: string) => Promise<void>`

### 1D. Frontend — wire callbacks in parent components

Update all three parents that use `FilesPanel` / `FileExplorer`:

- **`WorkspaceIdeView.tsx`** — add `handleCreateEntry`, `handleRenameEntry`, `handleDeleteEntry` that call the API and refresh the directory
- **`IdeRoute.tsx`** — same
- **`MobileIdeView.tsx`** — same (the context menu works on mobile too since it's click-based, not right-click)

Also update `FilesPanel.tsx` to pass through the new props.

## Feature 2: Full File Path in Editor Header

### 2A. Modify `FileEditorPanel.tsx`

**Multi-tab mode (tab bar)**: Change the tab label from `tab.path.split("/").pop()` to show the full path. Use `select-none` removal so users can select/copy it. Constrain width with `max-w-[200px]` so tabs don't overflow, with truncation.

**Single-file mode (mobile)**: Show full path instead of just filename.
